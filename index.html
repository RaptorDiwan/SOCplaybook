Key Questions to Ask on Every Alert
1
What is the alert?
Alert category, severity, rule triggered — understand what is being detected.
2
Who / What is involved?
Source host, destination, user account, process — identify all entities.
3
When did it happen?
Timestamp — business hours or off-hours? First occurrence or repeated?
4
Is there any context?
What was happening before and after? Any related alerts or events?
5
Is it malicious or benign?
Use OSINT, hash checks, and behavior to determine true/false positive.
6
What action is needed?
Close / Monitor / Contain / Escalate — always document your decision.
🎯 L1 Goal: Identify if the alert is a False Positive (FP) or needs escalation to L2.
#ActionTool / CommandNotes
Phase 1 — Alert Validation: "Is the alert real and relevant?"
1Read the alert carefullySIEM consoleIdentify alert name, severity, source host, user, timestamp, and rule triggered.
2Check for duplicates / noiseSIEM searchIs this alert firing repeatedly from same host/user? Might be a noisy rule.
3Validate contextSIEM (log view)Check event details — IP, hostname, logon type, command line, parent process.
4Check rule logicSIEM rule referenceUnderstand what this detection means — e.g., "Anomalous process execution."
Phase 2 — File / Process Validation: "Is the artifact legitimate?"
5Verify file signaturePowerShell → Get-AuthenticodeSignature "file_path"Signed and valid? If yes, likely safe.
6Check file hash reputationVirusTotal / HybridAnalysis0 detections = probably FP. Many hits = malicious.
7Validate file pathSIEM / EDR / endpointLegit: C:\Windows, C:\Program Files — Suspicious: AppData\Temp
8Check parent processSIEM / Sysmon Event ID 1If spawned by legitimate parent (e.g., explorer.exe), usually normal.
9Confirm process owner / userSIEM / Windows Event ID 4688SYSTEM or service account = normal; domain user = investigate.
Phase 3 — Behavioral Correlation: "Any signs of compromise?"
10Check related events (before & after)SIEM timeline / EDRAny suspicious commands, network calls, or registry changes?
11Look for network indicatorsEDR / Firewall logsOutbound connections after process creation?
12Review similar alerts from same host/userSIEM searchPersistent activity → possible compromise.
Phase 4 — Documentation & Handover
13Document all findingsSIEM ticketRecord alert details, actions, hashes, screenshots, and reasoning.
14Make decision—Close as FP (with justification) OR Escalate to L2 (with summary).
15Add whitelist / tuning suggestionSIEM tuning / detection feedbackSuggest suppression for similar false positives if legitimate.
L1 Decision Shortcuts
✅ File is signed + clean hash + expected path + known parent process → Mark as False Positive. Close.
⚠️ File is unsigned OR unknown hash → Do not close. Escalate to L2.
⚠️ Correlated activity looks suspicious (unusual network call, chained behavior) → Escalate to L2.
🔴 Any signs of persistence, lateral movement, or data exfil → Escalate immediately.
🎯 L2 Goal: Deeply investigate escalated alerts to determine if they are malicious, benign, or need further hunting.
#ActionTool / CommandResult / Notes
Phase 1 — Context Review: "What exactly happened?"
1Review L1 notes & SIEM dataIncident TicketConfirm alert details, affected host, user, and time. Reproduce event query if needed.
2Understand detection rule logicSIEM rule definitionKnow what behavior the rule detects (e.g., registry modification, PowerShell anomaly).
3Re-collect raw eventsSIEM / EDR queryPull 15–30 min window before & after the event for full context.
4Identify all affected entitiesSIEM searchHost, user, process, file, registry key, IP, domain.
Phase 2 — File / Process Deep Validation: "What is this binary doing?"
5Re-check signature & hashPowerShell Get-FileHash, PEStudio, VirusTotalConfirm integrity and reputation.
6Extract metadata & stringsPEStudio / Strings / ExifToolLook for suspicious imports, URLs, commands.
7Behavioral sandboxing (if unknown)Any.Run, HybridAnalysisObserve network calls, file writes, persistence.
8Check persistence locationsAutoruns / Registry / Task SchedulerHKLM\SYSTEM\CurrentControlSet\Services\, Run keys.
9Correlate parent-child treeEDR process treeConfirm expected chain (e.g., services.exe → legitimate driver).
10Correlate system logsSysmon ID 1/7045, Windows 4688/4697Validate installation method.
Phase 3 — Host & Network Correlation: "Evidence of compromise?"
11Host triageEDR / Memory dumpLook for unusual processes, DLL injections, network sockets.
12Network correlationFirewall / Proxy / DNS logsAny external connections or C2 patterns?
13Cross-host correlationSIEM query by hash / filenameSee if same artifact exists on multiple systems.
14Threat Intel enrichmentMISP / VirusTotal / internal feedMatch IoCs (domains, hashes, IPs) with known campaigns.
Phase 4 — Risk Assessment & Decision
15Assess intent and impactAnalyst judgmentDoes activity align with business software or attacker behavior?
16Check policy / change recordsCMDB / IT Change MgmtIf legitimate IT maintenance, mark benign.
17Decide classification—False Positive / Benign / Suspicious / Confirmed Malicious
18Recommend actions—Containment, isolation, block hash/domain, tune rule.
Phase 5 — Reporting & Knowledge Sharing
19Document investigationCase Mgmt / SOAR ticketInclude screenshots, IoCs, reasoning, results.
20Feed results to Threat Intel / L3TI platform / Ticket escalationShare new indicators or detection gaps.
21Tune detection if FPSIEM rule exceptionAdd condition or whitelist to reduce noise.
🎯 L3 Goal: Root-cause analysis, threat hunting, reverse engineering, and strengthening detection logic.
#ActionTool / PlatformResult / Notes
1Review escalated L2 case & confirm hypothesisIncident TicketRead L2 summary, confirm data completeness, identify gaps.
2Deep correlation across environmentSIEM, EDR, Network telemetryIdentify lateral movement, related hosts, user sessions.
3Reverse engineering / advanced static analysisGhidra, IDA Free, dnSpyUnderstand executable purpose, APIs, obfuscation.
4Memory and forensic analysisVolatility, KAPE, FTK ImagerLook for in-memory implants, injected threads, persistence.
5Advanced network behavior analysisZeek, Wireshark, Suricata, Threat Intel feedsDetect beaconing, C2 communications, anomalies.
6Correlate MITRE ATT&CK techniquesMITRE ATT&CK NavigatorMap all identified TTPs to tactics for reporting.
7Create detection logic & validate hypothesisRule CreationDevelop and test new detections or hunts for missed activity.
8Perform threat huntingSIEM, EDR, Sysmon datasetsSearch for indicators, similar activity across enterprise.
9Containment / Response coordinationSOAR, EDR isolation commandsWork with IR team to contain affected endpoints/users.
10Post-incident report & detection feedbackInternal knowledge base / Threat Intel platformDocument root cause, update detections, share IOCs.
